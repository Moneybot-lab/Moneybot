# Historical Validation Contract

MoneyBot Page 6 produces an immutable dataset manifest, evaluation metrics, explicit promotion gates, and a rollout recommendation before any candidate is broadly enabled.

## Artifacts

- `decision_outcomes_snapshot.json` — materialized realized outcomes used as the current operational input.
- `historical_validation_report.json` — versioned Page 6 report generated by `scripts/page6_historical_validation_report.py`.
- `dataset_manifest.v1` — dataset ID, source, row count, date range, adjustment method, point-in-time declaration, delisted-security declaration, and SHA-256 checksum.
- `promotion_gates.v1` — machine-readable pass/fail result for every blocking or warning gate.

A model artifact should record the dataset ID and checksum used to train or validate it. A checksum change means a different dataset, even when the human-readable dataset ID was reused.

## Metrics

The validator reports:

- Brier score and expected calibration error.
- BUY and SELL precision/recall.
- Gross and friction-adjusted average return.
- Worst return and maximum adverse excursion when supplied.
- Recommendation churn.
- Profile override rate and privacy-safe profile bucket counts.
- Stale-data and fallback/source-mode rates.

The report accepts precomputed rows so future Massive flat-file and point-in-time pipelines can use the same promotion contract without changing rollout logic.

## Promotion and rollback

Promotion is blocked when any blocking gate fails. Default blockers include insufficient evaluated rows, unacceptable calibration, negative net return, excessive churn, excessive stale/fallback rates, adverse-excursion regression, or incomplete licensing/privacy review.

The safe default is always `hold_shadow`. Operators should only increase allowlist or percentage rollout when `promotion_ready=true`, subgroup review is complete, and Page 4/Page 5 production reliability gates also pass.

## Generate and inspect

```bash
python scripts/page6_historical_validation_report.py \
  --outcomes data/decision_outcomes_snapshot.json \
  --output data/historical_validation_report.json \
  --dataset-id moneybot-outcomes-YYYY-MM-DD \
  --licensing-review-complete \
  --privacy-review-complete
```

The weekly model refresh now generates the Page 6 report after outcome materialization. The protected endpoint `GET /api/historical-validation` requires `X-Daily-Ops-Token`. `/api/model-health` exposes only the rollout summary and artifact freshness.

## Current limitation

The validation contract and reporting path are implemented, but full Massive flat-file ingestion, point-in-time fundamentals, delisted-security coverage, and true walk-forward backtests remain production data-pipeline work. They must be completed before Page 6 can be marked fully operational.
